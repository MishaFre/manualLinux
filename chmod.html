Comando CHMOD

Descripción:

chmod es un comando que se utiliza en sistemas operativos basados en Unix y Unix-like (como Linux y macOS) para cambiar los permisos de acceso a archivos y directorios.
El nombre chmod es una abreviatura de "change mode". Los permisos de acceso determinan qué usuarios o grupos pueden leer, escribir o ejecutar un archivo.

Opciones:

                
                    -c, --changes         Reporta solo cuando se realiza un cambio.

                    -R, --recursive       Esta opción permite cambiar los permisos de un directorio y todos sus archivos y subdirectorios 
                                          de manera recursiva. Es útil cuando se desea aplicar los mismos permisos a todo el contenido de 
                                          un directorio de forma recursiva.

                    -x, --remove-exec     Elimina los permisos de ejecución de un archivo.

                    -v, --verbose         Muestra un mensaje por cada fichero procesado

Ejemplos de uso:

                
                    
                    chmod 744 archivo.txt           Otorga permisos de lectura, escritura y ejecución al propietario del archivo (7), 
                                                    mientras que el grupo y otros usuarios solo tienen permiso de lectura (4),
                                                    todo esto para el archivo archivo.txt.
                    
                    chmod -c ug+x ejemplo.txt       Añade permisos de ejecución (+x) para el propietario (u) y el grupo (g) del archivo ejemplo.txt.
                                                    mostrando un mensaje sólo si se realizan cambios en los permisos del archivo.

                    chmod -R 755 directorio/        Cambiará los permisos de manera recursiva en el directorio directorio/ y todos sus archivos y 
                                                    subdirectorios. Los permisos 755 significan que el propietario tendrá permisos de lectura, escritura 
                                                    y ejecución (7), mientras que el grupo y otros usuarios solo tendrán permisos de lectura y ejecución (5).
                                            
                    chmod -x script.sh              Supongamos que tenemos el archivo “script.sh” y tiene permisos de lectura, escritura y ejecución para el propietario. 
                                                    Con este comando se eliminan los permisos de ejecución para el propietario. Con chmod go-x script.sh eliminaríamos permisos 
                                                    de ejecución para el grupo y otros.

                    chmod -v u=rw cuentas.txt       Cambia los permisos del usuario en el archivo cuentas.txt, y envia un mensaje tras cambiar los permisos
